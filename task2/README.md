### Задача
Превратить осенний пейзаж с кленами в летний и летний в осенний. На вход подаются Photo1.jpg (осень) и Photo2.jpg (лето). На выходе - Summer.jpg и Autumn.jpg.

### Метод
Алгоритм работает в цветовом пространстве HSV. В отличие от RGB, HSV разделяет цвет на три части: Hue (оттенок, 0-180 в OpenCV), Saturation (насыщенность) и Value (яркость). Это позволяет менять оттенок листвы, не трогая текстуру и освещение.

Суть подхода: осенняя листва клена занимает диапазон Hue 0-35 и 170-180 (красный, оранжевый, желтый), а летняя - 30-85 (зеленый). Программа находит пиксели листвы по оттенку, строит маску и линейно переносит их Hue из одного диапазона в другой. Потом подкручивает насыщенность и яркость, чтобы результат выглядел естественно. Остальные части сцены (небо, земля, стволы) не затрагиваются.

### Структура алгоритма

Алгоритм состоит из пяти шагов, каждый реализован отдельной функцией.

**1. Построение бинарной маски (_build_raw_mask).**
Картинка переводится из BGR в HSV. Для каждого пикселя проверяется, попадает ли его Hue в заданный диапазон. Пиксели с низкой насыщенностью (S < sat_min) или яркостью (V < val_min) отбрасываются - это убирает небо, землю, стволы и тени.

**2. Сглаживание маски (_refine_mask).**
Бинарная маска чистится двумя фильтрами OpenCV: MORPH_CLOSE заполняет мелкие дырки внутри кроны, MORPH_OPEN убирает случайный шум вокруг. Затем маска переводится во float32 [0..1] и размывается гауссом (21x21). Это дает плавные границы без резких переходов.

**3. Перенос оттенка (_remap_hue).**
Для каждого пикселя под маской считается его позиция t от 0 до 1 внутри исходного диапазона Hue. Потом t линейно отображается в целевой диапазон: new_hue = dst_center - dst_spread + t * 2 * dst_spread. Это сохраняет разброс оттенков - листва не становится одноцветной.

**4. Коррекция насыщенности и яркости (_apply_sv_factors).**
Каналы S и V домножаются на коэффициенты из конфига. Для осень->лето: sat_factor=0.60, val_factor=0.90 (летняя зелень приглушенная). Для лето->осень: sat_factor=1.35, val_factor=1.08 (осенние краски яркие и сочные). Коррекция применяется только в области маски.

**5. Смешивание с оригиналом (_alpha_blend).**
Результат смешивается с исходной картинкой по мягкой маске: original + (converted - original) * mask. Где маска равна 0 - пиксели не меняются, где 1 - полностью заменяются, между ними - плавный переход.

### Конфигурация

Все параметры собраны в RecolorConfig (frozen dataclass): диапазоны исходных оттенков, целевой центр и разброс, пороги маски, коэффициенты S/V. Два готовых конфига - AUTUMN_TO_SUMMER и SUMMER_TO_AUTUMN - заданы как константы модуля.

### Достоинства
- Работает за O(n) по числу пикселей, не нужны GPU и нейросети.
- Сохраняет текстуру, освещение и детали - меняется только цвет.
- Мягкая маска дает плавные переходы без артефактов на границах.
- Линейный перенос Hue сохраняет разнообразие оттенков внутри кроны.
- Конфигурация в dataclass - легко добавить новые типы преобразований.

### Недостатки
- Объекты с похожим оттенком (зеленая одежда, оранжевая крыша) тоже перекрасятся.
- Не учитывается разница в освещении сцены между сезонами; коррекция V компенсирует это лишь частично.
- Пороги подобраны под типичные фото с кленами. На снимках с другим балансом белого или растительностью параметры надо подстраивать.

### Запуск
```bash
python season_convert.py
```

Photo1.jpg и Photo2.jpg должны лежать в текущей директории. Результат - Summer.jpg и Autumn.jpg.
